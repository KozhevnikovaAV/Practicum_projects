# ПОСТРОЕНИЕ ДАШБОРДА С ПОМОЩЬЮ "Tableau" С ИТОГАМИ ОПРОСА ДЛЯ ТЕЛЕКОММУНИКАЦИОННОЙ КОМПАНИИ

## ЗАДАЧА:   
1. С помощью SQL-запросов выгрузить поля:
   - идентификатор клиента;
   - количество дней «жизни» клиента;
   - поле хранит информацию о том, является ли клиент новым;
   - возраст;
   - пол;
   - тип операционной системы;
   - тип устройства;
   - страна проживания;
   - город проживания;
   - возрастной сегмент;
   - сегмент по объёму потребляемого трафика;
   - сегмент по количеству дней «жизни»;   
   - оценка клиента в NPS-опросе;   
   - поле хранит информацию о том, к какой группе относится оценка клиента в опросе.

2. Подготовить дашборд с итогами опроса и ответить на вопросы:
- Как распределены участники опроса по возрасту и полу?
- Каких пользователей больше: новых или старых?
- Пользователи из каких городов активнее участвовали в опросе?
- Какие группы пользователей наиболее лояльны к сервису? Какие менее?
- Какой общий NPS среди всех опрошенных?
- Как можно описать клиентов, которые относятся к группе cторонников?

3. Подготовить презентацию.

## ДАННЫЕ:   
файл (telecomm_csi.db) с базой данных хранит таблицы:   
* таблица, хранящая информацию о клиентах (user):
  - идентификатор клиента (первичный ключ таблицы);   
  - количество дней «жизни» клиента;
  - возраст клиента в годах;
  - пол клиента (1 – женщина, 0 – мужчина);
  - тип операционной системы;
  - тип устройства;
  - идентификатор домашнего региона клиента (внешний ключ, отсылающий к таблице-справочнику территорий);   
  - идентификатор возрастного сегмента клиента (внешний ключ, отсылающий к таблице с данными о возрастных сегментах клиентов);
  - идентификатор сегмента клиента по объёму потребляемого трафика в месяц (внешний ключ, отсылающий к таблице с данными о выделяемых сегментах по объёму потребляемого трафика);
  - идентификатор сегмента клиента по количеству месяцев «жизни» (внешний ключ, отсылающий к таблице с данными о выделяемых сегментах по количеству месяцев «жизни» клиента);
  - оценка клиента в NPS-опросе (от 1 до 10).
* таблица-справочник территорий, в которых телеком-компания оказывает услуги (location):
  - идентификатор записи (первичный ключ);
  - страна;
  - город.
* таблица, хранящая информацию о возрастных сегментах клиентов (age_segment):
  - идентификатор сегмента (первичный ключ);
  - минимальная граница сегмента;
  - максимальная граница сегмента;
  - название сегмента.
* таблица, хранящая данные о выделяемых сегментах по объёму потребляемого трафика (traffic_segment):
  - идентификатор сегмента (первичный ключ);
  - минимальная граница сегмента;
  - максимальная граница сегмента;
  - название сегмента.
* таблица, хранящая информацию о выделяемых сегментах по количеству месяцев «жизни» клиента (lifetime_segment):
  - идентификатор сегмента (первичный ключ);
  - минимальная граница сегмента;
  - максимальная граница сегмента;
  - название сегмента.
